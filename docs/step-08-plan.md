# 第八步计划：权限细化与多实例安全

第七步完成了备份、恢复、脱敏导出和运维状态页。第八步继续补生产环境下多人使用和多实例部署时的安全边界。

## 目标

- 管理员账号可以在页面内维护。
- 高风险动作按角色授权。
- 多实例运行时后台 worker 不重复执行同一类任务。
- 运行配置可以只读展示，便于排查生产问题。

## 后端任务

- 新增管理员管理接口：列表、创建、禁用、重置密码、修改自己的密码。
- 增加角色授权中间件：viewer 只读，operator 可执行同步/健康检查/补池，admin 可管理账号和导出。
- 增加 `worker_locks` 表，通过 MySQL 行锁或唯一键租约保护周期任务。
- 新增只读配置接口，返回脱敏后的关键运行配置和启动时间。

## 前端任务

- 新增“管理员”页面：账号列表、创建账号、重置密码、禁用账号。
- 高风险按钮根据角色禁用，并显示清晰的状态。
- 运维状态页补充 worker 锁状态和只读配置。

## 完成标准

- viewer 账号无法执行写操作。
- operator 可执行日常运维，但无法管理管理员账号。
- admin 可完成账号管理和导出。
- 启动两个 KeyHub 实例时，同一 worker 周期内只会有一个实例执行任务。
